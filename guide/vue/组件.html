# 组件

# 组件

Vue中的组件化开发就是把网页重复的代码抽离出来，封装成一个个可以复用的组件，然后将这些视图组件拼接在一起就构成了我们的一个完整的应用

# 组件的组成部分

组件就是对局部视图的封装

  • html
  • css
  • js
    • data
    • methods

# 全局注册

全局组件
Vue.component("ComponentGlobal", {
    // 一个组件内只能由一个根节点
    template: `<div><h1 @click="check">头部组件-{{name}}-{{index}}</h1><h1 @click="check">头部组件-{{name}}-{{index}}</h1></div>`,
    data() {
        return {
            name: "全局组件",
            index: 0
        }
    },
    methods: {
        check() {
            this.index++
        }
    }
})

# 局部注册

// 局部组件 const localComponent = { template: <div><h1></h1></div>, data() { return { massage: "局部组件" } }, methods: {} }

new Vue({
    el: "#app",
    data: {
        arr: [1, 2, 3, 4, 5, 6, 7]
    },
    components: {
        "local-component": localComponent
    }
})

# ⭐全局组件和局部组件的区别：

​ 全局组件和局部组件的区别就是，局部组件只需要在vue实例上面挂载，而全局组件需要写在Vue.component('组件名称',{配置项})方法上面

# 组件化注意事项

  1. 组件命名规范
  2. 组件内部的template必须只有一个根节点
  3. ⭐组件的data必须是一个函数，并且函数体返回一个对象，每个组件都可以维护一个对象的独立拷贝，如果没有这条规则的话，操作数据会影响所有的组件实例
  4. 组件可以理解为一个特殊的vue实例，不需要手动实例化，只管理自己的template模版
  5. 组件与组件之间是互相独立的，可以配置自己的一些配置项 data ， methods， 等等
  6. 思想： 组件自己管理自己，不影响别人

# 组件化的问题

  1. 全局定义的组件要求组件名称不能重复
  2. 字符串模版缺乏语法高亮，在html多行的时候写起来很麻烦
  3. 不支持css
  4. 模块化问题：commonjs 和 ESmodules
