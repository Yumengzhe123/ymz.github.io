    # 计算属性和watch监听

    当结果需要复用的时候，可以使用计算属性来实现

    计算属性写法上像函数，但是本质上是属性

    计算属性中 必须写return

    # 计算属性和方法的区别：

    ​ 计算属性是基于它们的响应式依赖进行缓存的

    ​ 只有在相关响应式依赖发生改变，才会重新计算结果，如果没有改变，那么计算属性会立即返回之前的计算结果相比之下，每当重新渲染的时候，调用函数总会再次执行

    # watch监听：

    deep:是否进行深度监听

    immediate:开始监听的时候，立即执行一次回调函数

    computed和watch的区别：

    1.computed支持缓存，watch不支持缓存，数据变，直接触发相应的操作

    2.computed不支持异步，当计算属性内有异步操作时无效，无法监听数据的变化，watch支持异步

    3.computed没有参数，基于计算属性的响应式依赖进行缓存，watch接受两个参数

    4.computed中有get()和set()方法，watch只是监听到数据变化，执行函数
