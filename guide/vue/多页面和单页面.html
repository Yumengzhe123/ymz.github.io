# 多页面如何实现

  1. 根据页面去划分整个项目
  2. 浏览器自身实现页面切换
  3. 页面跳转本质上是从服务端请求html

# 多页面的特点

  1. 页面切换简单
  2. 对SEO搜索引擎优化较好
  3. 开发比较困难---难以实现模块化，应用入口多

# 单页面

  1. 通过js模拟页面的切换效果，并且与url地址栏挂钩
  2. 根据视图划分的应用

# 单页面的特点

  1. 对seo搜索引擎不太友好
  2. 所有的视图和逻辑，仅仅使用一个页面完成
  3. 开发相对简单（应用入口只有一个）

# 前端开发的痛点

  1. 开发时模块化不好处理
  2. 需要处理的文件类型很多（js,css,html,图片）
  3. 需要考虑部署在服务器，以及运行在浏览器的问题

# webpack

  1. 认为所有的前端资源都是模块
  2. 所有的静态资源都可以使用commonjs模块化的方式进行组织
  3. 提供了可以把任意资源转换成js的能力
  4. 可以把所有的模块按照引用逻辑组织在一个文件（js），供浏览器使用

# 前端框架

VUE：尤雨溪 2014年开发的前端框架 MVVM 学习成本低 数据的双向绑定 React：Facebook推出的前端框架 使用jsX 一切以js为核心 Angular：谷歌推出的前端框架 google维护的 功能复杂，学习路线陡峭

# MVVM和 MVC

# MVC开发模式

M：model模型 V：View视图 C：Controller控制器 view层接受用户动作，传递指令给controller controller完成业务逻辑，要求Model改变状态 model将新的数据发送给View，用户得到了反馈

# MVVM开发模式

M：model模型 V：View视图 VM：ViewModel（视图模型） Controller修改为ViewModel

view层接受用户请求：DOM事件，AJax ViewModel监听View的请求和model层的数据状态变化 ViewModel和model 进行了数据的双向绑定，model层监听ViewModel的变化

# ⭐v-for 和v-if 的优先级

v-for优先级高，不建议放在同一个标签（组件）上使用
先循环所有的组件之后，再进行判断，会造成性能浪费
解决办法：
    1. 可以使用计算属性来实现（先处理数据，再渲染）
    2. 可以将 v-if 置于外层元素 (或 <template>) 上
